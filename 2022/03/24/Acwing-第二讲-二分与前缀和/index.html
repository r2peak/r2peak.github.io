第二讲 二分与前缀和

  1. 确定一个区间，使得目标值一定在区间中。
  2. 找一个性质，满足
  • 性质具有二段性（前半段满足，后半段不满足。中间没有缺失的地方。）
  • 答案是二段性的分界点。

整数二分

*下面两种二分的模板，可以解决所有二分的问题。 *

核心在第四点，懂了代码就没问题了。

总结一下：
**二分x的左端点，r = mid;**时，int mid = l + r >> 1;
**二分x的右端点，l = mid;**时，int mid = l + r + 1 >> 1;

这是前人的经验总结，无需纠结，记住就好。我们做事要”知其然，也要知其所以然“，上面是”知其然“的部分，下面就是“知其所以然”的部分了。

下面是帮助理解这个知识点的内容，真正敲代码还得靠上面的思路。

特点：在数轴上的取值是离散的。答案分两大类：
一个是红色区间的右端点，一个是绿色区间的左端点。

第一类：ans是红色区间的右端点。

由于整数有下取整，因此M = (L + R) / 2要变成M = (L + R + 1) / 2

注意：这作为二分的第一个模板，它的标志是什么呢？就是我们在写的时候，不需要真的去把图画出来，去考虑是红色的右端点，我们只需要观察一下，判断条件后面是不是L = M，如果判断条件后面写了这个的话，我们在算中点M的时候，补上加1就行。

为什么要补上1呢？这里是为了解决一个边界问题，如果不补1，那么当L = R - 1的时候，M = （L + R) / 2 = L，此if(M 红)成立，L = M = L，这个while循环，执行过一次之后，L 和 R 没有发生任何变化，还是会进行下次循环，然后每一次循环 L 和 R 都不变。因此就死循环了。

不加1，实际上算的是(L + R) / 2的下取整，
加1，算的是(L + R) / 2的上取整。

加上1之后，M = (L + R + 1) / 2 = R然后执行L = R，区间就只有一个数了，就停下来了，不会出现死循环。

第二类：ans是绿色区间的左端点。

注意：同理，见到条件是R = M。不需要补上加1.

这里是不需要加1的，如何验证呢？只考虑一下边界情况即可。

L = R - 1
M = (L + R) / 2 = L
第一种情况，M绿
R = M = L // 区间只有一个数了，就会退出，不会死循环。
第二种情况，M红
L = M + 1 = L + 1 = R // （由第一行L = R - 1），也会判断正确，不会死循环。

实数二分

是稠密的，非常好写，没整数二分那么麻烦。

789.实数范围

这题特殊在排序后存在重复的数字，如果不存在重复数字的话，哪个模板都一样。

Tips:

下面是找左端点的情况：条件为q[mid] >= x（看图很好理解），同理，之后找右端点的条件为：q[mid] <= x


二分的时候，数组按照 x 的大小被分成了两个部分，二分就是查找两个部分的分界点。当 l == r 时就是找到了分界点。

实数二分

实数二分有个好处，M可以恰好取到中点，因为实数是稠密的。所以，就不要考虑中点属于左边还是右边了。

整数二分的点是离散的，其边界是当区间里只有一个数时停止，否则继续二分。

实数二分当区间长度足够小的时候停止，写成while (R - L > 1e-6)

Tips:

计算mid时，不是整除了，是严格除法。（不移位，直接/）

数的三次方根

有单调性，一定是可以二分的。但能二分的不一定具有单调性。也就是说，没有单调性也可以二分。

三次方根是具有单调性的，

if(m * m * m >= x) R = M // 说明太大了，缩小点：
else L = M; // 太小了，就放大点。

题目要求，取小数后6位，一般我们多取两位，取到8位以保证我们的精度是ok的。

结果输出左端点还是右端点都是ok的，因为精度足够。
printf("%lf\n", l); // 这样默认保留6位小数
要想保留4位的话，printf("%.4lf\n", l);即可。
